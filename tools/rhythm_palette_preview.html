"Add rhythm" picks the first-unused palette index from the natural cyclic position outward (see PluginEditor.cpp:99), so for a fresh patch the slot-0 default (Green) is followed by index 1 (Red). The complaint: Red is visually strong for the second slot — wants a softer second pick. Reorder the array above to whichever sequence reads as "calm → bold" for the typical 2-3 rhythm patch and let me know the new index order. Hex values + names are stable so the new order is just an array shuffle in rhythmPalette[]; existing presets store the palette INDEX in r0_colour so changing the order will re-tint existing presets — usually fine since the palette is small, but worth being aware of.
